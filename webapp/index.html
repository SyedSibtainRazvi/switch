What is context0?

AI coding sessions have no memory between tools. context0 stores compact checkpoints in local SQLite, scoped to your git repo and branch — so your task state follows your code, not your tool.

Scoped

Context key is repo_root + branch, so each branch stays isolated.

Local

No cloud dependency. Data is saved at ~/.context0/context0.db by default.

Fast

Save, resume, and log are lightweight CLI operations built for active dev loops.

Quickstart

Best flow: install context0, add MCP + a rule file, then let the AI save and resume context. Say “save context” or “I’m switching” when you change tools.

  1. Install

    curl -fsSL https://raw.githubusercontent.com/SyedSibtainRazvi/context0/main/install.sh | sh

  2. Add MCP and rule files

    context0 init-rules

    Then configure context0 mcp-server in your tool (see MCP setup below).

  3. Use it

    When you open a session, the agent calls get_context and resumes. When you say “save context” or “I’m switching”, the agent calls save_context with a summary. No manual CLI needed.

  4. CLI fallback

    context0 save --done "wired auth middleware" --next "fix integration tests"

    Resume: context0 resume or context0 resume --json

Commands

save

Write a checkpoint for the current repo + branch.

context0 save \
  --done "implemented OAuth flow" \
  --next "add refresh token logic" \
  --blockers "waiting on API key" \
  --tests "cargo test auth::" \
  --files src/auth.rs src/middleware.rs

resume

Print the latest checkpoint for the current repo + branch.

context0 resume
context0 resume --json

log

Show recent checkpoints in reverse chronological order.

context0 log --limit 20

mcp-server

Run MCP over stdio so tools can read/write context directly.

context0 mcp-server

init-rules

Install agent rule files for Claude Code, Cursor, and Codex. Run once per project.

context0 init-rules

clear

Delete all checkpoints for the current repo + branch.

context0 clear

MCP setup

Add context0 mcp-server to your editor/tool MCP config and use tools: get_context, save_context, and list_context. 

{
  "mcpServers": {
    "context0": {
      "command": "context0",
      "args": ["mcp-server"]
    }
  }
}

File: ~/.claude/settings.json

{
  "mcpServers": {
    "context0": {
      "command": "context0",
      "args": ["mcp-server"]
    }
  }
}

File: .cursor/mcp.json (project) or ~/.cursor/mcp.json (global)

Storage and scope

Default database

~/.context0/context0.db

Custom database

context0 --db /path/to/custom.db resume
